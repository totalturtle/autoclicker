# 안드로이드 오토클리커 앱 기능 초안

## 개요
Android Accessibility Service를 활용한 자동 터치/클릭 앱

---

## 핵심 기능

### 1. 클릭 포인트 설정
- 좌표 수동 입력 (X, Y)
- **터치로 좌표 직접 선택** ✅ 구현됨
  - 포인트 추가 다이얼로그에서 "터치로 시작 좌표 선택" 버튼 클릭
  - 반투명 전체화면 오버레이(CoordPickerService) 표시
  - 원하는 위치 터치 → 앱 복귀 + 다이얼로그 자동 재오픈 후 X/Y 자동 입력
  - 스와이프 끝 좌표 / 트리거 확인 좌표도 동일 방식으로 터치 선택 가능
- 다중 포인트 등록 (최대 50개)
- 포인트별 이름/라벨 지정

### 2. 클릭 시퀀스 (순서 제어)
- 등록된 포인트를 순서대로 반복 클릭
- 포인트 순서 드래그앤드롭으로 변경 ✅ 구현됨
- 반복 횟수 설정 (1회 ~ 무한) ✅ 구현됨
- 전체 시퀀스 반복

### 3. 타이밍 제어
- 각 클릭 간 딜레이 설정 (ms 단위) ✅ 구현됨
- 포인트별 개별 딜레이 설정 ✅ 구현됨
- **딜레이 전체 적용 버튼** ✅ 구현됨 — 메인 딜레이 값을 모든 포인트에 일괄 적용
- 포인트별 랜덤 오차 설정 (탐지 방지용) ✅ 구현됨
  - 각 포인트 딜레이 기준 ±N ms 범위 내 랜덤 (0이면 비활성)
  - 예: 딜레이 1000ms, 오차 100ms → 900~1100ms 랜덤
  - 포인트별 설정값 우선, 미설정 시 전역 딜레이 사용
- **오차 전체 적용 버튼** ✅ 구현됨 — 메인 오차 값을 모든 포인트에 일괄 적용
- 클릭 지속 시간(Long Press) 설정 ✅ 구현됨

### 3-1. 포인트 반복
- **포인트별 반복 횟수** ✅ 구현됨 — 각 포인트를 N번 연속 실행 후 다음 포인트로 이동
  - 반복 횟수 1 = 단일 실행 (기본값)
  - 예: 포인트 A(반복 10회) → 포인트 B(반복 1회) → 루프 반복
  - 포인트 간 딜레이는 반복 중에도 동일하게 적용
  - 다이얼로그의 "반복 횟수" 필드로 설정
- **반복 적용 버튼** ✅ 구현됨 — 메인 앱에서 전체 시퀀스 반복 횟수 즉시 저장 (딜레이 전체 적용과 별개)

### 4. 제스처 지원
- 단순 탭 (Tap) ✅ 구현됨
- 롱 프레스 (Long Press) ✅ 구현됨
- 스와이프 (Swipe) ✅ 구현됨
  - **끝점 마커 `N-1`** ✅ 구현됨 — 스와이프 포인트 설정 시 끝점 마커(빨간 원, X 표시, 점선 테두리) 자동 생성
  - 오버레이 편집 모드에서 `N-1` 마커 드래그 → endX/endY 즉시 저장
  - **꺾인 스와이프(다중 경유점)** ✅ 구현됨
    - 오버레이 다이얼로그 "꺾임 추가" 버튼으로 `N-2`, `N-3` … 경유 마커 추가
    - 각 마커 드래그로 경로 설정, `GestureDescription` 다중 세그먼트 Path로 실행
    - "꺾임 삭제"로 마지막 경유점 제거, 관련 마커 즉시 제거
  - 오버레이 다이얼로그에서 끝 좌표 입력 필드 대신 마커 드래그 방식으로 대체 (스와이프 지속시간만 입력)
- 드래그 (Drag) — 미구현
- 핀치 줌 인/아웃 — 미구현

### 5. 시작/정지 제어
- 플로팅 버튼으로 언제든 시작/정지 ✅ 구현됨
  - ACTION_DOWN 기반 터치 리스너 사용 (제스처 주입 중 ACTION_UP 누락 방지)
  - 400ms 디바운스로 중복 입력 방지
- 물리 버튼(볼륨 다운 키)으로 시작/정지 토글 ✅ 구현됨
- 알림바 Quick Tile로 제어 ✅ 구현됨 (타일 시각 상태 반영 포함)
- 횟수 조건 충족 시 자동 정지 ✅ 구현됨
- 메인 앱에서 시작/정지 버튼 제거 (오버레이 패널로 통합)

### 6. 프로필 관리 ✅ 구현됨
- 다중 named 프로필 저장/불러오기 (SharedPreferences)
- 프로필 이름 지정
- 프로필 내보내기/가져오기 (JSON, SAF 파일 피커)
- 앱별 프로필 자동 연동 (특정 앱 포그라운드 전환 시 자동 로드)

### 7. 조건부 실행 (스마트 트리거) ✅ 구현됨
- 특정 색상 감지 시 클릭 실행 ✅ 구현됨 (API 30+ / Android 11 이상, `android:canTakeScreenshot="true"` 필수)
  - 포인트 실행 전 지정 좌표의 색상을 스크린샷으로 체크 → 일치 시 탭, 불일치 시 아래 동작
  - **SKIP 모드**: 불일치 시 즉시 해당 포인트 건너뜀 → 다음 포인트로 진행
  - **WAIT_RETRY 모드**: 불일치 시 `재시도 간격(ms)` × `최대 재시도 횟수`만큼 반복 체크 → 그래도 없으면 스킵
  - 허용 오차(tolerance 0~255) 설정 가능
  - 트리거는 무한반복 기능이 아님 — 전체 반복 횟수(repeatCount)와 독립 동작
  - 딜레이After는 탭 여부와 무관하게 항상 적용됨
  - **색상 피커(스포이트)** ✅ 구현됨
    - 오버레이 포인트 설정 다이얼로그 → "색상 피커" 버튼
    - CoordPickerService 스포이트 오버레이 즉시 표시 (사전 스크린샷 불필요)
    - 스포이트 끝점(✕)이 손가락보다 90dp 위에 표시 → 손가락에 가려지지 않음
    - 드래그하면 끝점 위치 실시간 좌표 칩 표시 `(x, y)`
    - 손가락을 떼면 끝점 좌표의 픽셀 색상을 접근성 서비스 takeScreenshot()으로 추출
    - **"탭 위치도 동일하게" 토글** ✅ 구현됨 (기본: OFF)
      - 스포이트로 찍으면 항상 색상값 추출 + 트리거 확인 위치를 스포이트 좌표로 설정 (고정)
      - ON: 탭 위치(포인트 X/Y)도 스포이트 좌표로 이동 (편의 옵션)
      - OFF: 탭 위치는 그대로, 트리거 확인 위치만 스포이트 좌표로 설정
    - 색상 HEX 필드 옆 실시간 컬러 프리뷰 표시
    - API 30 미만 기기는 미지원 (Int.MIN_VALUE 취소 신호 처리)
  - 트리거 확인 위치와 탭 위치를 각각 다르게 설정 가능 (`cbTriggerSameAsPoint` 해제 시)
- 특정 이미지/아이콘 인식 후 클릭 (OpenCV) — 미구현
- 텍스트 인식(OCR) 후 조건 분기 (ML Kit) — 미구현

### 8. 화면 오버레이 UI ✅ 구현됨
- **플로팅 컨트롤 패널** (항상 화면 위에 표시, 권한 확보 시 앱 시작과 함께 자동 실행)
  - `· · ·` 핸들로 패널 드래그 이동
  - ⚙ 설정 버튼 → 메인 앱 열기
  - **편집 버튼** → 편집 모드 토글 (편집 중일 때 +/- 버튼 표시)
  - **+ 버튼** → 화면에 드래그 가능한 포인트 마커 생성 (번호 순)
  - **- 버튼** → 마지막 포인트 마커 삭제
  - 👁 눈 아이콘 → 마커 표시/숨김 토글
  - ▶ / ■ 시작 / 정지
  - 클릭 횟수 실시간 표시
  - **초기화 버튼** → 모든 포인트 삭제 (실행 중이면 먼저 정지)
  - **파워 버튼** (빨간색) → 플로팅 서비스 종료
- **드래그 가능한 포인트 마커** ✅ 구현됨
  - + 누를 때마다 화면에 번호가 표시된 마커 생성
  - 마커를 드래그해 원하는 위치로 이동 → 좌표 자동 저장
  - 마커 탭 → 세부 설정 팝업 (오버레이 다이얼로그)
    - 제스처 변경 (탭/롱프레스/스와이프)
    - 딜레이, 랜덤 오차(±ms), 레이블, 포인트 반복 횟수, 트리거 설정
    - 개별 포인트 삭제
  - **스와이프 끝점 마커 (`N-1`)** ✅ 구현됨
    - 스와이프 포인트 저장 시 끝점 마커 자동 생성 (빨간 원, X 표시, 점선 테두리)
    - 편집 모드에서 끝점 마커 드래그 → endX/endY 즉시 저장
    - 끝점 마커 탭 → 해당 포인트 설정 다이얼로그 열림
  - 👁 눈 아이콘으로 마커 표시/숨김 토글 (시작점/끝점 마커 모두 적용)
  - 자동 클릭 실행 중에는 마커 자동 숨김, 정지 후 복원
- 자동 클릭 실행 중에도 패널 유지

## 보조 기능

### 9. 통계 및 로그 ✅ 구현됨
- 세션별 총 클릭 횟수 / 스킵 횟수 / 경과 시간 기록
- 인메모리 로그 (타임스탬프 + 이벤트 메시지)
- 로그 파일 내보내기 (SAF .txt 저장)

### 10. 설정 ✅ 구현됨
- 다크/라이트/시스템 기본 테마 선택
- 앱 시작 시 저장된 테마 즉시 적용 (Application 클래스)
- Accessibility Service 권한 안내 및 바로가기
- 앱 버전 정보 표시
- 언어: 한국어/영어 (OS 언어 자동 감지)

### 11. 첫 실행 가이드 ✅ 구현됨 (UX 개선)
- 최초 실행 시 권한 설정 단계별 안내 팝업
- 접근성 권한 버튼 클릭 시 상세 안내 다이얼로그
  ("설치된 앱 → AutoClicker 자동 클릭 → 스위치 ON")
- 오버레이 권한 버튼 클릭 시 상세 안내 다이얼로그

## 권한 요구사항
| 권한 | 용도 |
|------|------|
| `BIND_ACCESSIBILITY_SERVICE` | 자동 터치 실행 핵심 권한 |
| `SYSTEM_ALERT_WINDOW` | 플로팅 오버레이 표시 |
| `FOREGROUND_SERVICE` | 백그라운드 실행 유지 |
| `FOREGROUND_SERVICE_SPECIAL_USE` | 포그라운드 서비스 타입 |
| `RECEIVE_BOOT_COMPLETED` | 부팅 후 자동 시작 (미구현) |

## 기술 스택
- **언어**: Kotlin
- **최소 SDK**: API 26 (Android 8.0)
- **타겟 SDK**: API 36
- **UI**: XML + ViewBinding
- **서비스**: AccessibilityService, ForegroundService
- **색상 감지**: AccessibilityService.takeScreenshot() (API 30+)
- **이미지 인식**: 미구현 (OpenCV 검토)
- **OCR**: 미구현 (ML Kit 검토)
- **저장**: SharedPreferences + JSON 파일 (SAF)
- **테마**: AppCompatDelegate DayNight

## 화면 구성

```
[메인 화면]
 ├─ 권한 상태 카드 (접근성 / 오버레이)
 ├─ 빠른 시작/정지 카드 (볼륨키 토글 스위치, QS Tile 안내)
 ├─ 클릭 포인트 카드
 │   ├─ 포인트 목록 (RecyclerView, 드래그 순서변경)
 │   ├─ 포인트 추가 버튼
 │   ├─ 눈 아이콘 (마커 표시/숨김 토글)
 │   ├─ 공통 딜레이 입력
 │   └─ 반복 횟수 입력
 ├─ 실행 상태 카드 (상태 텍스트, 클릭 횟수)
 ├─ 유틸리티 버튼 (프로필 관리 | 통계/로그 | 설정)
 └─ 시작 / 정지 버튼

[포인트 추가 다이얼로그]
 ├─ 제스처 선택 (탭 / 롱프레스 / 스와이프)
 ├─ 좌표 / 끝좌표 입력
 ├─ 지속시간 / 딜레이(ms) / 랜덤 오차(±ms) 입력
 └─ 색상 트리거 설정 (선택)
     ├─ 확인 좌표 X, Y
     ├─ 목표 색상 (#RRGGBB)
     ├─ 허용 오차
     └─ 불일치 시 동작 (스킵 / 재시도)

[플로팅 오버레이]
 ├─ ▶ 시작 / ■ 정지 버튼 (드래그 이동 가능)
 └─ 현재 클릭 횟수 텍스트

[프로필 관리]
 ├─ 현재 설정 저장 버튼
 ├─ 파일에서 가져오기 버튼
 └─ 프로필 목록 (불러오기 / 내보내기 / 앱연동 / 삭제)

[통계 및 로그]
 ├─ 세션 시작 시각 / 경과 시간
 ├─ 총 클릭 수 / 스킵 수
 ├─ 로그 내보내기 버튼
 └─ 타임스탬프 로그 (monospace)

[설정]
 ├─ 테마 선택 (시스템 / 라이트 / 다크)
 ├─ 접근성 서비스 바로가기
 └─ 앱 버전 정보
```
